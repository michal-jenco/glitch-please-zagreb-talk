## SEGMENT 2 — LIVE DEMO (12:00 – 27:00)

> *Keep the phone in one hand, the clicker in the other. Your voice is the soundtrack — don't let the app be the whole performance.*

### 12:00 – 14:00 · The first wow

**Do this on screen:**
1. Launch Glitch Please!
2. Load a dramatic source image from the pre-seeded gallery (pick whichever feels right in the room).
3. Pick **VHS** or **Color Gate** from the algorithm list.
4. Hit Glitch. Show the grid of variants.

**Say:**
- "37 algorithms, grouped into families — Specialists, Classics, Wave Riders, Deep Glitch, Spacetime, Decay, Channel Dominance."
- "Don't worry about the names. Worry about the feel."
- Point at one result: "This one I've never seen before. Every tap makes new ones."

### 14:00 – 17:00 · Palette as first-class citizen

**Do this on screen:**
1. Open palette editor.
2. Extract palette from the source image. Show swatches.
3. Generate random palette. Re-glitch with it.
4. Show how dramatically the result changes.

**Say:**
- "Most filter apps give you the filter. You tweak sliders. You don't touch the colour logic."
- "Here, the **palette defines the aesthetic**. Same algorithm, swap the palette — completely different piece."
- "You can extract from the source, generate random, or build one by hand."

### 17:00 – 20:00 · Results grid + "use result as source"

**Do this on screen:**
1. Generate a grid of variants.
2. Lock 2–3 favourites (show the lock icon).
3. Pick one, tap "use result as source."
4. Glitch again. Show evolution.

**Say:**
- "This is the move this app is built around. **Recursive glitching.**"
- "Pick a result. Feed it back in. Glitch again. Pick again. Feed back."
- "Every iteration layers on the last. You get shapes that no single-pass filter can reach."
- Tap a result to open Image Details: "And every image remembers exactly how it was made. Algorithm, palette, parameters. So an art form that's usually un-reproducible — here you can reproduce it, or walk away from it on purpose."

### 20:00 – 23:00 · Live editor

**Do this on screen:**
1. Open the live editor for VHS or Muse.
2. Scrub sliders. Let the audience see pixels move in real time.

**Say:**
- "This is running on a phone, in real time, no render-wait. One line of technical honesty: it's using lookup tables and chunking the pixel work across CPU cores with Kotlin coroutines. That's it. No magic."
- "For me as an artist this is the big thing — exploration, not commitment. I can *play*."

### 23:00 – 25:30 · Video loop export

**Do this on screen:**
1. Configure a progressive glitch loop (parameters evolve frame-to-frame).
2. Export. Show the result loop.

**Say:**
- "Glitch isn't just still images. It's kinetic. The app exports **video loops** where the parameters evolve frame by frame."
- "This takes glitch out of the gallery and into your phone feed."

### 27:00 – 28:30 · One code moment [slide 17]

> *Switch projector input back to laptop. Slide 17 shows the ORIGINAL algorithm — the first glitch you ever coded — with the mod256 wrap lines highlighted.*

**Say:**
- "I promised intro-level app dev, so — one screen of code. Don't panic."
- "This is called **ORIGINAL** in the app. It's the first glitch algorithm I ever coded — sitting in PyCharm in 2025. And it's still one of the signature looks of Glitch Please!"
- Point at the highlighted lines (`mod256` block):
  - "For every pixel, I take its red, green, and blue values. I shift each one by a chunk of math based on where the pixel is in the image. I add a random nudge. Then I **wrap the result at 256** — if a channel goes over 255, it loops back to 0."
- *(Pause.)*
- "That wrap — that's the glitch. When a byte overflows and loops around, you get the rainbow shimmer, the hard color jumps, the feeling that something broke."
- "No AI, no neural net. Just byte overflow, running on every pixel."
- "If you write any code — this is reachable. If you don't — now you know the spell."

> *Click to slide 18.*

## If things go wrong

- **Mirror drops:** Plug in HDMI, don't explain, keep going. Fall back to the backup screen recording.
- **App crashes mid-demo:** Laugh. Say "see, it's still glitch art." Relaunch. This is literally on-brand.
- **Running long at minute 20:** Skip the video loop (23:00–25:30) and go straight to the code moment.
- **Running short:** Extend Q&A. Don't pad the demo.
- **Nervous blank in the story:** The sentence "Two tracks, separate rooms, for a decade" gets you unstuck every time.
